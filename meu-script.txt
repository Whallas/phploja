baixar o composer.phar ou composer-setup.exe (windows)

criar o composer.json
    definir as dependências do projeto
    require: smarty, phpmailer
    vendor-dir: diretório dos pacotes de terceiros
ou
composer.init

outras diretivas para o composer.json:
    require-dev: phpunit etc...

apache - configurar openSSL no projeto (httpd-vhosts.conf)

instalar composer
C:\xampp\php\php.exe composer.phar install
composer install
ou
C:\xampp\php\php.exe composer.phar update
composer update

utilizando o smarty:
criar arquivo .tpl, fazer o template
no php chamar o smarty, criar suas variaveis e iniciar.

Model:
    É onde ficam as classes do projeto, são utilizadas pelo controller

Aqui é feito a reescrista de URL com um apache conf: .htaccess;
Com ele é possivel obter uma request mais suja utilizando-se uma mais amigável no navegador, junto com isso tem-se a classe Rotas que gerencia os parametros.
Basta criar o arquivo e não precisa reiniciar o servidor.
. Criei uma classe Config para parametrizar informações do projeto que são usadas em vários lugares, muito útil para manutenção e por exemplo, mudança de hosts.

Controller:
    Controla os models e as views. Nesse projeto cada model tem um controller próprio.

View:
    Basicamente é onde ficam os Templates, que correspondem cada um a um Model e controller, respectivamente, porém nem todos os models tem interface.
